8.4. Support vector machine (hard margin)#

The perceptron algorithm provides us a way of finding \(w\) and \(b\) that separates the data points. However, note that there could be many solutions. The solution it finds will depend on the initial values chosen for \(w\) and \(b\). Both Figure 1a and 1b shows valid solution.

If there are multiple solutions of the problem, then we should select the one that acheives minimum generalization error. Intuitively, we would prefer the classfier in Figure 1b since it has a large margin, in other words, the smallest distance between the hyperplane and any of the samples is larger than the one in Figure 1a. This means that it is less likely to make misclassification on the testing dataset.

How can we find such classifiers? We present a geometric formulation below.

8.4.1. Geometric margin#

To find the classifier that has a large margin, we need to first define margin mathematically. To do so, recall that the smallest distance from a point \(x_i\) to a hyperplane, denoted as \(\gamma_i\), is given by

\[\gamma_i=\frac{|w^Tx_i+b|}{||w||} = \frac{y_i(w^Tx_i+b)}{||w||}\]

The geometric margin \(\gamma\) is then the smallest among these, namely

\[\gamma = \min_{i\in [n]} \gamma_i\]

We want to make \(\gamma_i\) as large as possible, subject to the constraint that all data points are correctly classified. This can be framed as an optimization problem

\[\max_{w, b, \gamma} \gamma \hspace{5mm}\text{subj to} \hspace{3mm} y_i(w^Tx_i+b)\geq 0\]

Note that the above optimization problem is equivalent to

\[\max_{w, b, \gamma} \gamma \hspace{5mm}\text{subj to} \hspace{3mm} \frac{y_i(w^Tx_i+b)}{||w||}\geq \gamma\]

By a change of variables \(\gamma\mapsto \frac{\gamma}{||w||}\), we have

\[\max_{w, b, \gamma} \frac{\gamma}{||w||_2} \hspace{5mm}\text{subj to} \hspace{3mm} y_i(w^Tx_i+b)\geq \gamma \tag{1}\]

Note that if \(w^*, b^*, \gamma^*\) is a solution to the above problem, then \(\alpha w^*,\alpha b^*, \alpha\gamma^*\) is again a solution. Therefore, we can fix \(\gamma=1\) to get a unique solution. This gives us

\[\max_{w, b, \gamma} \frac{1}{||w||_2} \hspace{5mm}\text{subj to} \hspace{3mm} y_i(w^Tx_i+b)\geq 1\]

This is equivalent to

\[\min_{w, b, \gamma} \frac{1}{2}||w||_2^2 \hspace{5mm}\text{subj to} \hspace{3mm} y_i(w^Tx_i+b)\geq 1 \tag{2}\]

Problem \((2)\) is the quadratic programming problem associated with hard-margin support vector machine. This is called hard-margin because we do not allow any misclassifications on the training dataset. In the next section, we present some nonlinear optimization theory that helps us to solve the optimization problem.

8.4.2. Review of nonlinear optimization#

Consider the optimization problem

\[\min_x f(x) \hspace{5mm}\text{subj to} \hspace{3mm} g_i(x)\leq 0\;\text{for}\;i\in [n]\]

Define the Lagrangian function as

\[\mathcal{L}(x, \alpha) = f(x) - \sum_{i=1}^n \alpha_ig_i(x)\]

Consider the primal problem

\[\theta_P(x) = \max_{\alpha\geq 0} \mathcal{L}(x, \alpha)\]

Note that

\[\begin{split} \begin{equation*} \theta_P(x)= \begin{cases} f(x)\hspace{10mm}\text{if}\;g_i(x)\leq 0\;\text{for}\;i\in [n]\\ +\infty \hspace{10mm}\text{otherwise} \end{cases} \end{equation*} \end{split}\]

In other words, \(\theta_P(x)\) is the same as \(f(x)\) if \(x\) is in the original feasible region. Therefore, the original optimization problem is equivalent as

\[p^*=\min_x \theta_P(x) = \min_x \max_{\alpha\geq 0} \mathcal{L}(x, \alpha)\]

Which is an unconstrained optimization problem. Now consider the dual problem

\[d^*=\max_{\alpha\geq 0} \min_x \mathcal{L}(x, \alpha)\]

The dual problem is much easier to solve compared to the primal problem since it is affine in \(\alpha\). However, by the standard minimax inequality, the dual problem only provides a lower bound to the primal problem

\[d^*\leq p^*\]

Fortunately, if Slater’s condition is satisfied, we have \(d^*=p^*\). This suggests that we can solve the dual problem instead. Moreover, the optimal solutions \((x^*, \alpha^*)\) must satisfy the Karush–Kuhn–Tucker conditions.

  1. Stationarity Condition:

\[\nabla_{x}\mathcal{L} = 0\]

  1. Primal feasibility condition:

\[g_i(x)\leq 0\;i=1,2,...,n\]

  1. Dual feasibility condition:

\[\alpha_i\geq 0\;i=1,2,...,n\]

  1. Complementarity slackness condition:

\[\alpha_ig_i(x)=0\;i=1,2,...,n\]

It turns out that the optimization problem for SVM satisfies Slater’s condition, therefore in the next section we show how to solve the dual problem.

8.4.3. Solving the SVM problem#

The goal is the solve the optimization problem

\[\min_{w,b} \frac{1}{2}||w||_2^2 \hspace{5mm} \text{subject to} \; y_i(w^Tx_i+b)\geq 1\]

Defining the Lagrangian gives us

\[ \mathcal{L}(w, b, \lambda) = \frac{1}{2}||w||_2^2 + \sum_{i=1}^n \lambda_i (1-y_i(w^Tx_i+b))\]

The primal problem is given by

\[p^*=\min_{w, b} \max_{\lambda\geq 0} \mathcal{L}(w, b, \lambda)\]

Since the data is separable, Slater’s condition holds and therefore, the optimization problem has zero duality gap. This suggests that we can instead solve the dual problem

\[d^*=\max_{\lambda\geq 0}\min_{w, b} \mathcal{L}(w, b, \lambda)\]

From the KKT conditions, we know that the optimal solutions \((w, b)\) satisfies

  1. Stationarity Condition:

\[\begin{split}\begin{equation*}\begin{cases} \nabla_{w}\mathcal{L} = w - \sum_{i=1}^n \alpha_i y_ix_i=0\\ \nabla_{b}\mathcal{L}=-\sum_{i=1}^n \alpha_i y_i=0 \end{cases}\end{equation*}\end{split}\]

  1. Primal feasibility condition:

\[y_i(w^Tx_i+b)\geq 1\;i=1,2,...,n\]

  1. Dual feasibility condition:

\[\alpha_i\geq 0\;i=1,2,...,n\]

  1. Complementarity slackness condition:

\[\alpha_i[y_i(w^Tx_i-1)]=0\;i=1,2,...,n\]

We can solve the inner maximization of the dual problem by plugging the stationary condition into the Lagrangian function, we have

\[\begin{split} \begin{align*} \mathcal{L}(w, b, \alpha) &= \frac{1}{2} \bigg(\sum_{i=1}^n \alpha_i y_ix_i\bigg)^T\bigg(\sum_{i=1}^n \alpha_i y_ix_i\bigg) - \sum_{i=1}^n \alpha_i\bigg[y_i\bigg(\bigg(\sum_{i=1}^n \alpha_i y_ix_i\bigg)^Tx_i\bigg)-1\bigg]\\ &= \frac{1}{2} \sum_{i=1}^n\sum_{j=1}^n \alpha_i\alpha_jy_iy_jx_i^Tx_j - \sum_{i=1}^n\sum_{j=1}^n \alpha_i\alpha_jy_iy_jx_i^Tx_j -b\sum_{i=1}^n \alpha_iy_i+\sum_{i=1}^n \alpha_i\\ &= -b\sum_{i=1}^n \alpha_iy_i+\sum_{i=1}^n \alpha_i - \frac{1}{2} \sum_{i=1}^n\sum_{j=1}^n \alpha_i\alpha_jy_iy_jx_i^Tx_j \end{align*}\\ \end{split}\]

Therefore, it remains solving the dual problem

\[\max_{\alpha\geq 0}\sum_{i=1}^n \alpha_i - \frac{1}{2} \sum_{i=1}^n\sum_{j=1}^n \alpha_i\alpha_jy_iy_jx_i^Tx_j\]

This can be solved using quadratic programming softwares.

import matplotlib.pyplot as plt
import numpy as np
import cvxopt.solvers

class supportVectorMachine:
    
    def __init__(self, D):
        self.w = np.random.randn(D)
        self.b = np.random.randn()
        self.alpha = None
    
    def fit(self, X, y):
        N = len(X)
        K = np.array([np.dot(X[i], X[j])
              for i in range(N)
              for j in range(N)]).reshape((N, N)) 
        P = cvxopt.matrix(np.outer(y, y) * K) 
        q = cvxopt.matrix(-1 * np.ones(N)) 
        A = cvxopt.matrix(y, (1, N)) 
        b = cvxopt.matrix(0.0) 
        G = cvxopt.matrix(np.diag(-1 * np.ones(N))) 
        h = cvxopt.matrix(np.zeros(N)) 
        solution = cvxopt.solvers.qp(P, q, G, h, A, b) 
        alpha = np.ravel(solution['x']) 
        self.w = np.array([alpha[i] * y[i] * X[i] for i in range(len(X))]).sum(axis=0)
        self.b = np.sum([y[i] - np.dot(w, X[i]) for i in range(len(X))]) / len(X)
        self.alpha = alpha
    
    def predict(self, X):
        prod = np.dot(X, self.b) + w
        y = np.where(prod > 0, 1, -1)
        return y

x1 = np.random.multivariate_normal([0.8, 0.8], np.diag([0.15, 0.15]), 30)
x2 = np.random.multivariate_normal([-0.8, -0.8], np.diag([0.15, 0.15]), 30)
y1 = np.ones(30) 
y2 = -np.ones(30)
X = np.vstack((x1, x2))
y = np.concatenate((y1, y2))
indices = np.arange(X.shape[0])
np.random.shuffle(indices)
X = X[indices]
y = y[indices]

svm = supportVectorMachine(2)
alpha = svm.fit(X, y)
