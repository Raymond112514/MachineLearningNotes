Perceptron algorithm

7.3. Perceptron algorithm#

Recall that given data \(\mathcal{D}=\{(x_i, y_i)\}_{i\in [n]}\), we want to find \(w, b\) such that

\[y_i(w^Tx_i+b)\geq 0\hspace{2mm}\text{for all}\;i\in [n]\tag{*}\]

The perceptron algorithm is an iterative algorithm that solves \((*)\). Note that it suffice discussing the case whether \(b=0\) since if \(b\neq 0\), then we can apply the algorithm to the augmented data points \([x_i, 1]\) and read the parameter asssociated with the constant term.

The algorithm starts by randomly initializing the weight \(w\). Next, for each \(i\), the algorithm computes \(y_i(w^Tx_i)\) and

  • If \(y_i(w^Tx_i)\geq 0\), then the algorithm correctly classified the \(i\)th data, thus no action is needed

  • If \(y_i(w^Tx_i)< 0\), then the algorithm misclassfied the \(i\)th data. This means that we need to update \(w\) in a way that increases the quantity \(y_i(w^Tx_i)\).

How should we increase \(w\)? Suppose we add \(\xi\) (to be determined) to \(w\), we want to choose \(\xi\) so that

\[y_i((w+\xi)^Tx_i)>y_i(w^Tx_i)\implies y_i\xi^Tx_i>0\]

Note that if we choose \(\xi = y_i x_i\), then

\[y_i\xi^Tx_i = y_i^2x_i^Tx_i = ||x_i||_2^2>0\]

This means that if we update \(w\) by adding \(y_ix_i\), then we make \(y_i(w^Tx_i)\) more positive. This is the idea behind perceptron algorithm. We then repeat this until \(y_i(w^Tx_i)\geq 0\) for all \(n\). The algorithm is summarized below.

import numpy as np
import matplotlib.pyplot as plt

x1 = np.random.multivariate_normal([0.8, 0.8], np.diag([0.15, 0.15]), 30)
x2 = np.random.multivariate_normal([-0.8, -0.8], np.diag([0.15, 0.15]), 30)
y1 = np.ones(30) 
y2 = -np.ones(30)

plt.scatter(x1[:, 0], x1[:, 1], color='blue')
plt.scatter(x2[:, 0], x2[:, 1], color='red')
plt.xlabel('X1')
plt.ylabel('X2')
plt.title('Scatter Plot of Data Points')
plt.xlim(-2, 2)
plt.ylim(-2, 2)
plt.show()

X = np.vstack((x1, x2))
y = np.concatenate((y1, y2))
indices = np.arange(X.shape[0])
np.random.shuffle(indices)
X = X[indices]
y = y[indices]

def perceptronAlgorithm(X, y):
    n, d = X.shape
    w = np.random.randn(d)
    terminate = False
    nit = 1
    while not terminate:
        terminate = True
        for i in range(n):
            if y[i] * np.dot(w, X[i]) < 0:
                w += y[i] * X[i]
                terminate = False
        nit += 1
    return w, nit

w, nit = perceptronAlgorithm(X, y)
print(f"Number of iterations needed: {nit}")

We now provide a proof of correctness of the perceptron algorithm. Assume that the dataset is separable by \(w^*\) with \(||w^*||_2=1\), let \(w_t\) be the algorithm updates at time \(t\). Moreover, let

\[R=\max_i ||x_i||_2\hspace{5mm}r=\min_i |\langle x_i, w^*\rangle|\]

The key ingredient towards the proof relies on the following lemmas.

Lemma 1. \(w_t^Tw^* \geq tr\)

Proof:
We proceed by induction. If \(w_0=0\), then the statement holds. Now suppose the statement is true for some \(t\), then $\( \begin{align*} w_{t+1}^Tw^* &= (w_t + y_i x_i)^Tw^*\\ &= w_t^Tw^* + y_i x_i^Tw^*\\ &\geq tr + r = (t+1)r \end{align*} \)$ This proves the claim.

Lemma 2. \(||w_t||\leq \sqrt{t} R\)

Proof:
We again proceed by induction. If \(w_0=0\), then the statement holds. Now suppose the statement is true for some \(t\), then $\( \begin{align*} ||w_{t+1}||^2 &= (w_t+y_ix_i)^T(w_t+y_ix_i)\\ &= ||w_t||^2 + ||x_i||^2 + 2y_i x_i^Tw_t\\ &\leq tR^2 + R^2 + 0 = (t+1)R^2 \end{align*} \)$ This proves the claim.

Now to prove that the perceptron algorithm terminates, note that if \(\theta_t\) is the angle formed by \(w_t\) and \(2^*\), then

\[\cos\theta_t = \frac{w_t^Tw^*}{||w_t||||w^*||} \geq \frac{tr}{\sqrt{t}R}\]

If the algorithm never terminates, then the RHS goes to \(\infty\). However, \(\cos\theta_t\) is bounded by \(1\), contradiction. Therefore, the algorithm must terminate in finite steps. With the inequality, we can also determine time to convergence.

\[ \frac{tr}{\sqrt{t}R}\leq 1\implies t\leq \frac{R^2}{r^2}\]
